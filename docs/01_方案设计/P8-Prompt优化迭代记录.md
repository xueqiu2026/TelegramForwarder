# Prompt B 优化记录

> 基于 2026-06-11 A/B 测试输出的人工审阅反馈
> 状态：收集中，持续更新

## 基线版本

Prompt B（跨市场情报简报），在沙盒中输出质量优于 Prompt A。以下为需要优化的具体问题。

---

## 已确认的优化项

### 1. 删除"可信度"标签
- **问题**：公开事实（财报、链上数据）标"可信度：高"是废话
- **改法**：只在传闻类信息前加 `【传闻】` 标签，事实类默认不标注
- **来源**：用户反馈 06-11 16:50

### 2. "影响资产"改为方向判断
- **问题**：当前只列板块名（AI 基建链、半导体...），没说利好还是利空
- **改法**：改成 `→ 利空 AI 基建链` 或 `→ 利好隐私赛道`，带明确方向
- **来源**：用户反馈 06-11 16:50

---

## 待确认的优化项（之前分析发现，尚未与用户讨论）

### 3. 机会雷达每条太冗长
- **问题**：每个标的写 50-80 字一大段，不符合"30 秒抓重点"
- **建议**：压缩到 1-2 行，格式 `标的 + 核心数据 | 风险提示`

### 4. 资金信号是文字墙
- **问题**：700 字一整段无法快速扫描
- **建议**：改成 bullet list，每条一个数据点

### 5. 缺少顶部一句话核心判断
- **建议**：报告开头加一行 `📍 一句话：xxx` 概括今天最核心结论

### 6. 内容跨板块重复
- **问题**：ETH 数据在机会雷达和资金信号中重复出现
- **建议**：同一数据点只出现在一个板块里

---

### 7. 禁用 AI 惯用句式
- **问题**：输出中大量使用 AI 写作标志性句式，一看就不是人写的
- **来源**：用户反馈 06-11 20:57
- **禁用清单**：
  - 不是...而是...
  - 值得注意的是 / 引人注目的是
  - 一方面...另一方面...
  - 不仅...更... / 不仅仅是...
  - 与其说...不如说...
  - 毫无疑问 / 显而易见 / 不可否认
  - 换句话说 / 总而言之 / 归根结底
  - 事实上 / 本质上 / 某种程度上
  - 这意味着 / 这表明
  - 令人惊讶的是
  - 不得不说
- **改法**：在 prompt 中加一条规则，明确列出禁用句式，要求用口语化、有个人风格的表达替代

## 写作素材 Prompt 测试反馈

### 测试结论
| 版本 | 定位 | 评价 |
|---|---|---|
| A（素材卡片） | 罗列正反观点、数据、金句 | ❌ 输出太像 AI，结构太工整 |
| B（推文工厂） | 直接出 5 条推文 + 2 个 Thread | ✅ 质量好但**不稳定**，多次运行结果差异大 |
| C（反共识猎手）| 打脸主流观点 | 待定 |
| D（信息差猎手）| 抢中文圈未覆盖的信息 | ✅ 不错 |

### 8. Prompt B 输出质量不稳定
- **问题**：同一输入多次运行，推文草稿质量波动大
- **来源**：用户反馈 06-11 21:12
- **可能原因**：
  - prompt 缺少示例锚定（few-shot），模型自由度过高
  - 没有明确"好推文"的标准（什么算犀利？什么算有钩子？）
  - 输入消息量大时，模型选题不稳定
- **优化方向**：
  - 加 1-2 条示例推文作为风格锚点
  - 增加选题优先级规则（如：有具体数字的 > 纯观点的 > 纯叙事的）

### 9. Prompt D（信息差猎手）获正面评价
- **来源**：用户反馈 06-11 21:12
- **后续**：可考虑 B + D 融合，日常用 B 产出，穿插 D 的信息差挖掘
