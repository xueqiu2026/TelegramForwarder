# P3 TelegramForwarder规则重构与AI总结训练沙盒设计方案

## 1. 背景与痛点

在 Batch 2 阶段，我们初步实现了 React 前端控制台。然而在实际使用与审计中发现了以下核心痛点：
1. **规则配置过载与繁琐**：单一频道转发规则堆叠了约 30 个配置项，用户在不需要高级功能（如 AI、总结、推送等）时，仍被迫面对全部配置，体验冗余且复杂。
2. **多时间点配置隐患**：定时总结的时间支持逗号分隔配置（例如 `"06:00,12:00,18:00,00:00"`），但直接让用户手打字符串极易因中英文逗号混淆、格式不规范而导致解析崩溃。
3. **AI 沙盒缺乏专业调优能力**：原 AI 沙盒仅支持“单条消息测试 AI 响应”，无法模拟定时总结所面临的“多条真实历史消息混合输入（消息流聚合）”场景，更无法对不同的总结 Prompt 进行对比及效果打分。

---

## 2. 详细设计方案

### 2.1 RulesPage 规则设置重设计（5 子 Tab + 条件隐藏）

详情面板由长平铺结构改为 5 个子 Tab 切换（`概览`、`消息处理`、`AI配置`、`过滤规则`、`推送&总结`），并应用**条件级联隐藏**：

*   **智能隐藏逻辑**：
    *   **AI 配置**：默认 `启用 AI 智能处理` 为关闭状态，其余大模型选择、Prompt、图片上传等子参数输入框全部隐藏。开启主开关后，这些参数平滑展开展示。
    *   **消息处理**：自定义模板（如用户信息模板、时间模板、原始链接模板）仅在其前置开关（如 `显示发送者` 等）开启时才显示。
    *   **推送与总结**：`启用总结` 未勾选时，隐藏时间与总结 Prompt；`启用推送` 未勾选时，隐藏 Apprise 推送通道列表和配置组件。

*   **一键预设模板**：
    顶部提供 `应用预设模板` 下拉菜单，内置典型业务场景模板：
    *   **纯消息直转**（关闭 AI、过滤、推送及自定义排版，以最快速度转发）。
    *   **AI 翻译与摘要转发**（一键开启 AI 处理并填入标准摘要 Prompt）。
    *   **仅 RSS 订阅源**（仅输出 RSS，不进行 Telegram 间转发）。

---

### 2.2 定时总结“时间徽标管理器”

为了彻底解决符号错误和格式错误，将文本输入框重写为**交互式时间时刻添加器**：

```
定时总结时间点: [ 06:00 × ]  [ 12:00 × ]  [ 18:00 × ]
[ 选择时间: 00:00 ]  [+ 添加时刻]
```

*   **交互逻辑**：
    1.  已选时刻以 Badge 徽章形式排列，点击 `×` 可将其从当前规则中删除。
    2.  下方提供一个时间选择器（或格式化输入框）和 `[+ 添加时刻]` 按钮。
    3.  点击添加时，前端会对输入进行合法性正则验证（如匹配 24 小时制 `^(0[0-9]|1[0-9]|2[0-3]):[0-5][0-9]$`）。
    4.  添加成功后，前端自动组装为英文逗号分隔的字符串（如 `"06:00,12:00,18:00"`）进行隐式自动同步保存。

### 2.3 AI 定时总结 Prompt 专业训练沙盒

将 AI 沙盒重构为面向“定时总结提示词调优”的专业训练场，包含三个核心组件：

#### 2.3.1 50 频道全局内容分布大屏
*   **功能**：沙盒顶部折叠卡片展示 50 多个频道的数据全景图（基于 `scratch/messages_dump.json` 约 3.6MB 的消息文本分析）。
*   **原理**：后端分析消息在各个分类主题（例如：DeFi Alpha、Airdrop、Meme 推荐、大盘分析、资讯新闻等）的占比，呈现饼图或统计表。
*   **价值**：帮助用户基于这 50 个频道的整体发帖特征与噪声占比，设计针对性的总结过滤 Prompt。

#### 2.3.2 真实历史消息流聚合器（Input Aggregator）
*   **功能**：免去用户手动粘贴。在沙盒左侧配置：
    *   `[选择频道 (可多选)]`
    *   `[设定时间跨度 (过去 24 小时 / 过去 3 天 / 过去 7 天)]`
*   **原理**：点击“生成测试数据”后，系统自动读取 JSON，提取选定频道在对应时间窗内的全部消息，并按时间线拼接为一段真实的**聚合消息流文本**作为 AI 总结的测试输入，完美还原定时总结任务的真实负载。

#### 2.3.3 提示词 A/B 双栏对照与评估
*   **双栏架构**：左右并列排布 `Prompt A` 和 `Prompt B` 两个独立编辑区（支持 Monaco Editor）。
*   **一键运行与评估**：对同一聚合消息流执行两个 Prompt 测试，并在下方并列展示运行结果。
*   **性能指标统计**：
    *   **运行耗时**（毫秒）。
    *   **生成 Token/字数估算**。
    *   **排版及格式对照**。
*   **调试成果应用**：调试满意的 Prompt 支持“一键导入沙盒”或“一键应用到所有频道规则”。

## 3. 后端新增 API 规划

为支持上述功能，后端 `console.py` 补充以下 3 个 API 端点：

1.  **`POST /api/rules/{id}/sync-to-all`**
    *   **说明**：将指定规则的所有配置字段复制同步到其他所有规则中。
    *   **行为**：数据库事务内操作，排除主键 (`id`)、聊天绑定 (`source_chat_id`, `target_chat_id`) 和创建时间，对其余字段（如 AI 开关、推送选项、消息格式等）执行批量 `update`。
2.  **`GET /api/sandbox/history-samples`**
    *   **说明**：获取沙盒消息流测试所需的频道列表和最新样本索引。
    *   **行为**：解析 `scratch/messages_dump.json`，返回频道名与 ID 映射。
3.  **`POST /api/sandbox/aggregate-messages`**
    *   **说明**：根据选定频道和时间区间聚合真实消息流作为沙盒输入。
    *   **行为**：后端读取 JSON，过滤时间戳，返回合并的消息流文本。
4.  **`POST /api/sandbox/analyze-channels`**
    *   **说明**：大模型全局分析 50 频道内容分布。
    *   **行为**：抽样消息，使用 LLM 分析内容类型占比，输出 Markdown 诊断报告。
