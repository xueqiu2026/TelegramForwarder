## 背景与追加需求

为了彻底解决 AI 沙盒测试、规则批量同步及 06:00 定时总结三者之间在 Prompt 提示词层面上的隐性冲突与心智不一致问题，我们在原核查方案的基础上进行专项重构，实现"3 个定时总结 Prompt 在前端完全可见、可编辑、可同步"。

### 2. 后端 API 与路由设计 [MODIFY] [console.py](file:///D:/Axiangmu/TelegramForwarder/rss/app/routes/console.py)

#### 2.1 规则详情接口 (`get_rule_detail` L413 / `update_rule` L463)
- **读取（L421-L457 返回字典）**：加入 `summary_prompt_b` 和 `summary_prompt_d` 反序列化。
- **写入（L472-L480 字段白名单数组）**：将 `summary_prompt_b` 和 `summary_prompt_d` 加入 `update_rule` 的显式白名单列表，否则前端保存时后端会静默忽略这两个字段。

#### 2.2 批量同步接口改造 (`sync_rule_to_all` L1088-L1110)
当前 `sync-to-all` 为全量覆盖模式，改造为支持选择性字段同步：
- **后端**：接收 `fields: string[]` 请求体参数。当 `fields` 非空时仅同步指定字段；为空或未传则保持原有全量覆盖行为（向后兼容）。
- **前端**：同步按钮弹出多选对话框，用户勾选需同步的字段组。

#### 2.3 新增沙盒 Prompt 应用接口
新增 `POST /sandbox/apply-prompt` 端点，用于在沙盒中一键将优化后的 Prompt 覆盖至全库所有规则的对应字段中。

#### 2.4 统一 AI 沙盒消息聚合拼装格式 (`aggregate_messages` L1193-L1241)
将沙盒中 `=== 频道: xxx ===` 的块状排版，改写为与 `summary_scheduler.py` L147-L151 中 100% 一致的逐行 `【频道名】时间 - 消息` 格式。
